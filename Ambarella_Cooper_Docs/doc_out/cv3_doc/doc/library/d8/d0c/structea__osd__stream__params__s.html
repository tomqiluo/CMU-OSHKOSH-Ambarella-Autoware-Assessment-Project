Detailed Description

The structure to specify the area parameters for the display on a stream.

Because of the latency delay in DSP, circular buffers are used to avoid the data conflict when updating the OSD areas. The default circle size of circular buffer in this module is 4 + DSP's encode_dummy_latency for each area. It's a typical safe value for all cases when the OSD updating frequency is no more than the stream's fps. Less value can be used to save the overlay or blur arbitrary buffer size if updating frequency is slower. If the OSD area will be changed more than one time, the circle size of circular buffer should be at least 2, because DSP will keep one buffer when encoding on each frame.

Either overlay_area_roi_group or overlay_buffer_size can decide how much the buffer will be used by this stream OSD. So only one of them should be provided. The same is for blur_arbitrary_area_roi_group and blur_arbitrary_buffer_size.
