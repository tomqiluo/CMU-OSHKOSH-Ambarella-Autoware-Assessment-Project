IN struct net_run_cfg *  net_run,
IN struct net_input_cfg *  net_in,
IN struct net_output_cfg *  net_out 
)

This API run network with network ID after call nnctrl_load_net

Parameters
net_idthe network ID
net_retthe network running result
net_runthe pointer to change the network cfg on-the-fly
net_inthe pointer to input of network. NULL: Use the previous input address. Valid Pointer: Network's input configuration. If the app allocates the input memory by itself, and the memory address is changed every time, the APP needs to specify this. Run-time update rotate-flip and udpate_pitch.
net_outthe pointer to output of network. NULL: Use the previous input address. Valid Pointer: Network's output configuration. If the app allocates the output memory by itself, and this memory address is changed every time, the App needs to specify this. Run-time update rotate-flip and udpate_pitch.
Returns
0 = success, -1 = error. -EINTR if it was interrupted by a signal; -EAGAIN if after call nnctrl_suspend_net()
Examples
test_nnctrl.c, and test_pvanet.cpp.

◆ nnctrl_suspend_net()

AMBA_API int nnctrl_suspend_net ( IN uint8_t  quit_all)

This API suspends the current network from running, enabling the pending network to enter the running state afterwards. If it is set to quit_all, all running and pending networks will be abandoned. Users can reschedule the network after calling this function.

Parameters
quit_allthe flag if quit all networks
Returns
0 = success, -1 = error.
