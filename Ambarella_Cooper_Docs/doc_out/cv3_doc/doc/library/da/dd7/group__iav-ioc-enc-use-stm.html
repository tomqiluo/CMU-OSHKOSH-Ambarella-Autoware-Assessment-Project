◆ IAV_IOC_APPLY_IDSP_ENC_SYNC_PROC

#define IAV_IOC_APPLY_IDSP_ENC_SYNC_PROC   IAVENC_IOW(IOC_FRAME_SYNC, u16)

This API can be invoked during ENCODING state to configure frame sync related parameters between IDSP (like DPTZ) and VDSP (like stream encode offset). The IDSP sync parameters (like DPTZ) are configured by IAV_IOC_CFG_VIDEO_PROC, while the VDSP sync parameters are configured by IAV_IOC_CFG_FRAME_SYNC_PROC. This API works when strm_sync_type is set to IAV_IDSP_ENC_SYNC for the VDSP configuration.

#define IAV_IOC_RELEASE_FRAMEDESC   IAVENC_IOWR(IOC_FRAMEDESC_CTRL, struct iav_framedesc *)

This API can be invoked during preview/encoding state to release the locked frames of bitstream. Currently, it's used for release the frame user queried and consumed before, IAV driver will decide to release the frame or not according to the frame match result, which refer to frame info like @size, @data_addr_offset, @enc_done_ts, @frame_num, etc.
