typedef int(* draw_256_colors_image_func_t) (uint8_t *buffer, int w, int h, int pitch, int rotate_type, void *arg)

The function prototype to draw a 256-colors image on the given buffer which fills the whole screen.

Simulator platform: Only support EA_DISPLAY_JPEG mode.

Parameters
bufferThe buffer to draw the image.
wThe width of the buffer.
hThe height of the buffer.
pitchThe row pitch of the buffer.
rotateThe rotate of the buffer, 0x00: clockwise 0, 0x01: clockwise 90, 0x06: clockwise 180, 0x07: clockwise 270.
argThe argument which is passed in by ea_display_set_256_colors_image_with_cb.
Returns
int EA_SUCCESS: OK, EA_FAIL: Error

AMBA_API ea_display_t * ea_display_new ( ea_display_type_t  type,
int  id,
int  features,
void *  param 
)

Create a display.

Simulator platform: Only support EA_DISPLAY_JPEG mode.

Parameters
typeA type in ea_display_type_e.
idChangeable meaning decided by type.

The stream ID (int), if type is EA_DISPLAY_STREAM, EA_DISPLAY_STREAM_SERVER or EA_DISPLAY_STREAM_CLIENT.

A value in ea_display_vout_id_e if type is EA_DISPLAY_VOUT.

A value in ea_tensor_color_mode_e if type is EA_DISPLAY_JPEG.

Parameters
featuresThe features in ea_display_feature_e to enable on the display in bit masks.
paramChangeable meaning decided by type.

::ea_display_params_t if type is EA_DISPLAY_STREAM, EA_DISPLAY_STREAM_SERVER.

If type is EA_DISPLAY_STREAM, params can be NULL to let the lib decide overlay_buffer_offset and overlay_buffer_size.

If type is EA_DISPLAY_STREAM, overlay_buffer_offset can be specified but overlay_buffer_size will be ignored.

If type is EA_DISPLAY_STREAM_SERVER, overlay_buffer_offset must be specified but overlay_buffer_size will be ignored.

The IP address of server (char *) if type is EA_DISPLAY_STREAM_CLIENT.

The folder path (const char *) to save the JPEG file if type is EA_DISPLAY_JPEG. The file will be saved in order of 0.jpg, 1.jpg, 2.jpg, etc.

Otherwise reserved and NULL can be used.

Returns
ea_display_t * The pointer which represents the display. NULL will be returned if failed.
Examples
test_eazyai_display.c.

◆ ea_display_set_256_colors_image()

AMBA_API int ea_display_set_256_colors_image ( ea_display_t *  display,
const char *  title,
float  x,
float  y,
float  w,
float  h,
ea_tensor_t *  img 
)

Add a bounding box with 256-colors image inside to draw on the display.

Simulator platform: Only support EA_DISPLAY_JPEG mode.

Other parammeters to draw the bounding box, like box color and border thickness, are set through ea_display_obj_params_s returned by ea_display_obj_params(). The color parameters are the index in the 256 colors table. The transparent color index can be set through ea_display_obj_params(display)->transparent_color. The transparency parameters will be 0 (transparent) inside no matter what is set. It's not thread-safe to draw a content on a display. But there is no competition among different displays.

Parameters
displayA pointer which represents a display.
titleTitle of the bounding box.
xHorizontal position of the upper left corner of the box. The value is related to ea_display_obj_params_s::obj_win_w.
yVertical position of the upper left corner of the box. The value is related to ea_display_obj_params_s::obj_win_h.
wWidth of the box involving border thickness. The value is related to ea_display_obj_params_s::obj_win_w.
hHeight of the box involving border thickness. The value is related to ea_display_obj_params_s::obj_win_h.
imgA tensor which holds the 256-colors image data in 1x1xHxW shape.
Returns
int EA_SUCCESS: OK, EA_FAIL: Error

AMBA_API int ea_display_set_textbox ( ea_display_t *  display,
const char *  text,
float  x,
float  y,
float  w,
float  h 
)

Add a textbox to draw on the display.

Simulator platform: Only support EA_DISPLAY_JPEG mode.

Other parammeters to draw the bounding box, like box color and border thickness, are set through ea_display_obj_params_s returned by ea_display_obj_params(). It's not thread-safe to draw a content on a display. But there is no competition among different displays.

The text is inside a box. The text may be drawn on multile lines according to the box width. To draw text without box appeared, set ea_display_obj_params_s::border_thickness to 0. It will return error if the box area can't hold the entire text content. The area of the text can be calculated by text length, ea_display_obj_params_s::font_size, ea_display_obj_params_s::font_width_ratio and ea_display_obj_params_s::line_spacing_ratio.

Parameters
displayA pointer which represents a display.
textText inside the box.
xHorizontal position of the upper left corner of the box. The value is related to ea_display_obj_params_s::obj_win_w.
yVertical position of the upper left corner of the box. The value is related to ea_display_obj_params_s::obj_win_h.
wWidth of the box involving border thickness. The value is related to ea_display_obj_params_s::obj_win_w.
hHeight of the box involving border thickness. The value is related to ea_display_obj_params_s::obj_win_h.
Returns
int EA_SUCCESS: OK, EA_FAIL: Error
