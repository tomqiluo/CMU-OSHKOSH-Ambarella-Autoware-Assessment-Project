◆ IAV_IOC_SET_OVERLAY_INSERT

#define IAV_IOC_SET_OVERLAY_INSERT   IAVENC_IOW(IOC_OVERLAY_INSERT, struct iav_overlay_insert *)

These API can be used to insert a selected picture into the video stream as an overlay. It is intended for the OSD overlay library use, and it is supported by H.264, H. 265 and MJPEG. Because the picture data is added to the video data before encoding, it is also referred to as "OSD blending".

It is the responsibility of the application to draw different overlay content in the memory partition. The application needs to call the "mmap" system in order to map the overlay memory partition before drawing the new content.

In CV5x implementation, the overlay is added to the source buffer. This means that if the two streams share the same source buffer, the OSD overlay will be affected. Refer to user guide documentation for details.
