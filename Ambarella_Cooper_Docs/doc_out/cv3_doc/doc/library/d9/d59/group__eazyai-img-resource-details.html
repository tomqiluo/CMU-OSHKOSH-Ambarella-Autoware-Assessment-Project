◆ ea_img_resource_hold_data()

AMBA_API int ea_img_resource_hold_data ( ea_img_resource_t *  img_resource,
ea_img_resource_data_t *  data 
)

Get and hold one frame of image data.

Simulator platform: Supports EA_RAW_FILE, EA_RAW_FILE_NV12, EA_RAW_FILE_FOLDER, EA_JPEG_FOLDER mode on simulator platform.

If the image resource type is EA_CANVAS, ea_img_resource_data_s::tensor_group will hold image data in order of canvas buffer, ME0 and ME1. If canvas is configured as prev type, there will be no ME0/ME1 data.

If the image resource type is EA_CANVAS_GROUP, ea_img_resource_data_s::tensor_group will hold image data in order of canvas buffer 0, ME0 and ME1, canvas buffer 1, ME0 and ME1, canvas buffer 2, ME0 and ME1, etc. If canvas is configured as prev type, there will be no ME0/ME1 data. The canvases in the group should come from the same channel.

If the image resource type is EA_PYRAMID, ea_img_resource_data_s::tensor_group will hold image data of each pyramid layer. If a layer is disabled, the tensor pointer will be NULL.

If the image resource type is EA_RAW_FILE, there will be only one tensor pointer in ea_img_resource_data_s::tensor_group, which holds the entire data in the file. Repeatedly calling will hold the same file content.

If the image resource type is EA_RAW_FILE_NV12, there will be only one tensor pointer in ea_img_resource_data_s::tensor_group, which holds the entire data in the file. Repeatedly calling will hold the same file content. The Y data is stored in data->tensor_group[0]. The UV data is stored in ea_tensor_related(data->tensor_group[0]).

If the image resource type is EA_RAW_FILE_FOLDER or EA_JPEG_FOLDER, there will be only one tensor pointer in ea_img_resource_data_s::tensor_group, which holds data of one file in the folder. The data of each other file will be held after each call of this function.

If the image resource type is EA_CANVAS or EA_PYRAMID, the color mode of image data will be EA_TENSOR_COLOR_MODE_YUV_NV12. For a image with size (W, H), if ::ea_tensor_related(tensor) is not NULL, the tensor holds Y data and the shape is 1x1xHxW, ea_tensor_related() is used to fetch the UV data and the shape is 1x1x0.5HxW. If ::ea_tensor_related(tensor) is NULL, the tensor holds YUV data and the shape is 1x1x1.5HxW.

If the image resource type is EA_RAW_FILE or EA_RAW_FILE_FOLDER, the color mode of image data will be the same as the original data.

If the image resource type is EA_JPEG_FOLDER, the color mode of image data will be EA_TENSOR_COLOR_MODE_BGR because it's read with OpenCV.

Parameters
img_resourceA pointer which represents a image resource.
data[out] Image resource data, including image data in tensors and DSP PTS.
Returns
int EA_SUCCESS: OK, EA_FAIL: Error
Examples
nn_input.c, and test_vproc_harris_live.cpp.

◆ ea_img_resource_new()

AMBA_API ea_img_resource_t * ea_img_resource_new ( ea_img_resource_type_t  type,
void *  from 
)

Create a image resource.

Simulator platform: Supports EA_RAW_FILE, EA_RAW_FILE_NV12, EA_RAW_FILE_FOLDER, EA_JPEG_FOLDER mode on simulator platform.

Image resource may be IAV canvas, IAV pyramid, raw binary files and JPEG files.

When reading a raw binary file, if ea_img_resource_set_raw_file_shape() is not used, the file name should have the shape information as [N]x[C]x[H]x[W]_*.bin, e.g. 1x3x480x640_dog.bin.

When reading a raw binary file which holds YUV NV12 data, if ea_img_resource_set_raw_file_shape() is not used, the file name should have the shape information as [N]x[C]x[H]x[W]_*.bin, e.g. 1x1x480x640_dog.bin. H and W are the shape of the Y data.

When reading a JPEG file, the file name should have the .jpg suffix.

When reading images from a folder, the reading is ordered by the file names.

Parameters
typeA type in ea_img_resource_type_e.
fromChangeable meaning decided by type.

Cavans ID (int) if type is EA_CANVAS.

Cavans ID map (unsigned int) if type is EA_CANVAS_GROUP.

Channel ID (int) if type is EA_PYRAMID.

File path (const char *) if type is EA_RAW_FILE or EA_RAW_FILE_NV12.

Folder path (const char *) if type is EA_RAW_FILE_FOLDER or EA_JPEG_FOLDER.

Returns
ea_img_resource_t * The pointer which represents the image resource. NULL will be returned if failed.
Examples
nn_input.c, test_vproc_harris_live.cpp, and test_vproc_render3d_live.cpp.

AMBA_API int ea_img_resource_set_file_info ( ea_img_resource_t *  img_resource,
const ea_img_resource_file_info_t *  info 
)

Set the file information for searching files and parsing the data in raw binary files.

By default, the filename extension is .jpg for image files, .bin for raw binary files. The shape of the raw files is parsed from the file name. The data format of the raw files is uint8.

Typical data format: uint8 (0,0,0,0), float32 (1,2,0,7)
