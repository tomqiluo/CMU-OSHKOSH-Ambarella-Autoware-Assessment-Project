ADVANCED - Encode Mode

1. Encode Mode

There are several encoding modes in the CV5x / CV72 chips that provide different functionalities and performances for various products. Although the CV5x / CV72 platform currently has fewer encoding modes than CV2x platform, additional modes are expected to be released in later versions. For both the CV5x / CV72 and CV2x chip series, the field of the encoding mode must be explicitly specified.

1.1 Comparison

The following table lists the differences in functions for encode mode comparison.

Table 1-1. Encode Mode Comparison.
Function Mode 0 Mode 1 Mode 5 Mode 127
IQ in low light Good Good Excellent No
Single region LDC Yes No Yes No
EIS Yes No Yes No
Multi-Region dewarp No Yes No No
HDR 2X Yes Yes Yes No
HDR 3X No No Yes No
Multi-VIN Yes No Yes No
Multiple YUV input Yes Yes Yes No
Single-Color privacy mask Yes Yes Yes No
Multi-Color privacy mask No No No No
Note
  • For mode 0, the DSP pipeline latency is lower than mode 5.
  • Compared with mode 0/1, mode 5 has better image quality with better MCTF effects for low light conditions.
  • Mode 0 and mode 1 has the same image quality.
  • For mode 127, it is VIN only mode, and it is normally used for RAW data capture purpose.

1.2 Mode 0: Normal ISO Mode

Table 1-2. Normal ISO Mode.
Feature
Description
Sensor Type CFA / YUV Input
Main Buffer Size Width <= 8192
Stream Number Up to 20 streams
Feature Included
  • All normal ISO features
  • Encoding
  • Overlay
  • Electronic image stabilization (EIS)
  • Single-Color privacy mask
  • Digital PTZ (type I and II)
  • Up to 3 layers SVC-T for H.264 and H.265
  • Capturing RAW / YUV / ME1 / ME0 data
  • High MPixel encoding (width > 1920)
  • Up to 4K 2X line interleaved HDR
  • MB level encode tools for H.264 and H.265
  • Frame sync mechanism
  • Lens distortion correction
  • Multi-VIN design
  • IDSP rotate / flip
Feature Not Included
  • Fisheye dewarping
  • Multi-Pass scalers
  • Fusion for RGB and IR channel
  • Fusion for RGB and thermal channel

1.3 Mode 1: Multi-Region Warp

Table 1-3. Multi-Region Warp.
Feature
Description
Sensor Type CFA / YUV Input
Main Buffer Size Width <= 4096
Stream Number Up to 20 streams
Feature Included
  • All advanced ISO features
  • Encoding
  • Overlay
  • Single-Color privacy mask
  • Up to 3 layers of SVC-T for H.264 and H.265
  • Captures RAW / YUV / ME1 / ME0 data
  • High megapixel encoding (width > 1920)
  • MB level encode tools for H.264 and H.265
  • Frame sync mechanism
  • Fisheye dewarping

Feature Not Included
  • Electronic image Stabilization (EIS)
  • 3X line interleaved HDR
  • Digital PTZ (type I and II)
  • Lens Distortion Correction
  • Fusion for RGB and IR channel
  • Fusion for RGB and thermal channel

Refer to ADVANCED - Multi-Region Dewarp for more details and examples.

2. Encode Mode Examples

Example 1: Normal ISO mode example

board# init.sh --na;modprobe imx274_mipi vinc_id=8 bus_id=3 // For CV5 or CV52
(Or board# init.sh --na;modprobe imx274_mipi vinc_id=8) // For CV72
board# test_aaa_service -a &
board# test_encode --resource-cfg /usr/share/ambarella/lua_scripts/cv5_vin0_1080p_linear.lua --vout-cfg /usr/share/ambarella/lua_scripts/vout_hdmi.lua --enc-mode 0 // For CV5 or CV52
(Or board# test_encode --resource-cfg /usr/share/ambarella/lua_scripts/cv72_vin0_1080p_linear.lua --vout-cfg /usr/share/ambarella/lua_scripts/vout_hdmi.lua --enc-mode 0) // For CV72
