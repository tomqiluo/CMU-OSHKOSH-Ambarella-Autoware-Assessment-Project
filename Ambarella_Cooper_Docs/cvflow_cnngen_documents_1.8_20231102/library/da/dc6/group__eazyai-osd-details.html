◆ ea_osd_area_current_buffer()

const AMBA_API ea_osd_area_buffer_t* ea_osd_area_current_buffer ( ea_osd_t *  osd,
int  local_area_id,
const ea_osd_roi_t *  dynamic_roi 
)

Get current buffer of the area.

The first calling of this function between the callings of ea_osd_refresh() will clear the buffer to EA_OSD_TRANSPARENT_COLOR_INDEX.

If an area with dynamic ROI is not touched by this API after ea_osd_refresh(), it will be cleared after the next ea_osd_refresh().

The area with dynamic ROI should only be touched once by this API before every new ea_osd_refresh(), or else the overlay buffer will be wasted because it's allocated in every call.

Parameters
osdThe OSD object.
local_area_idFor Stream, local_area_id is the area index counted in ea_osd_stream_params_t, the order is overlay areas first, dsp blur arbitrary areas later. For VOUT Screen, currently there's only one local_area_id which is 0.
roiThe dynamic ROI of the area in the OSD.

It should be NULL if ea_osd_stream_params_t::overlay_buffer_size or ea_osd_stream_params_t::blur_arbitrary_buffer_size for this area haven't been used in ea_osd_new(). Otherwise, it should be specified.

Returns
const ea_osd_area_buffer_t * The structure of the area buffer.

◆ ea_osd_new()

AMBA_API ea_osd_t* ea_osd_new ( ea_osd_type_t  type,
int  id,
const void *  params 
)

Create a OSD object.

If params is NULL, there will be one full screen area.

Parameters
typeA type in ea_osd_type_e.
idFor VOUT screen, it specifies the frame buffer file at /dev/fb[id]. For stream, it's stream ID.
paramsA pointer to ::ea_osd_screen_params_s or ea_osd_stream_params_s decided by type. It can be NULL, which means one full screen overlay area will be used. Otherwise, if overlay_area_num, blur_arbitrary_area_num and blur_box_area_num are all zeros, one full screen overlay area will be used as well.
Returns
ea_osd_t * The created OSD object.
Examples
case_ea_env.cpp, and case_ea_osd.cpp.
