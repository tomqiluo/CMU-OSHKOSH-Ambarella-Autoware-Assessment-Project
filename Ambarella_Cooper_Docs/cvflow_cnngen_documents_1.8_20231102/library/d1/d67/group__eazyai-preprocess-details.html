Crop and resize images from source to destination.

Simulator platform: Only support EA_CPU mode.

Only device = EA_VP and device = EA_DSP is supported by now.

For device = EA_DSP, only tensors with EA_TENSOR_COLOR_MODE_YUV_NV12 data are supported. The tensors in src should be from IAV query or created with ea_tensor_new_from_iav_anon_mem(). The tensors in dst should be created with ea_tensor_new_from_iav_anon_mem().

For device = EA_VP, tensors with EA_TENSOR_COLOR_MODE_YUV_NV12 data should hold the entire YUV data with shape 1x1x1.5HxW, and ::ea_tensor_related(tensor) is NULL.

The batch operation is supported. The batch is decided by dst and dst_num, the accumulation of N in the shape of each tensor. The valid number of ROI areas in roi_group is decided by batch. When the batch in src is less than the batch in dst, the last tensor in src is reused.

Due to the difference of CVflow ARCH, the resize operation on CV7x (Gen-3 CVflow ARCH) with EA_VP doesn't get binary-matched output by comparing to CV2, CV2x and CV5x (Gen-2 CVflow ARCH).

Parameters
srcA tensor pointer array which represents the source tensors.
src_numThe number of tensors in src.
dstA tensor pointer array which represents the destination tensors.
dst_numThe number of tensors in dst.
roi_groupA pointer which represents ROI areas. If NULL, the ROI area is the entire image in src.
color_modeThe color mode of the image data in src and dst. It's a type in ::ea_tensor_color_mode_s.
deviceA type in ea_device_e.
Returns
int EA_SUCCESS: OK, EA_FAIL: Error

Convert color and resize an image from source to destination.

Simulator platform: Only support EA_CPU mode.

The convert type EA_COLOR_BGR2GRAY and EA_COLOR_RGB2GRAY, only device = EA_CPU is supported, and handled by OpenCV. For other convert types, only device = EA_VP is supported by now. The batch operation is not supported, so the shape of tensors should be 1xCxHxW. The convert type EA_COLOR_YUV2RGB_NV12, EA_COLOR_YUV2BGR_NV12 and EA_COLOR_TRANSPOSE are accelarated by VP.

Due to the difference of CVflow ARCH, some operations on CV7x (Gen-3 CVflow ARCH) with EA_VP, such as EA_COLOR_YUV2BGR_NV12, don't get binary-matched output by comparing to CV2, CV2x and CV5x (Gen-2 CVflow ARCH).

Parameters
srcA pointer which represents the source tensor.
dstA pointer which represents the destination tensor.
typeA type in ea_color_convert_type_e.
deviceA type in ea_device_e.
Returns
int EA_SUCCESS: OK, EA_FAIL: Error
Examples
case_ea_preprocess.cpp, and test_eazyai.c.

Rotate and flip a tensor.

Only device = EA_VP is supported by now. Only the tensor with shape[EA_N]=1 is supported. Only support 8-bit data format. For the tensor of EA_TENSOR_COLOR_MODE_YUV_NV12, the src and dst tensors should have shape in the format of 1x1x1.5HxW.

Rotating EA_TENSOR_COLOR_MODE_YUV_NV12 with EA_VP is not supported on CV7x by now.

Parameters
srcA pointer which represents the source tensor.
dstA pointer which represents the destination tensor.
flip_bitmapThe bitmap which indicates the combined operations of rotate and flip. BIT0: rotate 90 clockwise, BIT1: horizontal flip, BIT2: vertical flip, BIT3: channel flip.
color_modeThe color mode of the tensor in src. It's a type in ::ea_tensor_color_mode_s.
deviceA type in ea_device_e.
Returns
int EA_SUCCESS: OK, EA_FAIL: Error

◆ ea_warp_new()

AMBA_API ea_warp_t* ea_warp_new ( const ea_warp_params_t *  params,
ea_device_t  device 
)

Create a warp context.

Simulator platform: API not supported.

It only supports warp with EA_VP now, and only handles BGR, RGB and gray scale images. And EA_VP can only do the grid-level warp. The grid spacing should be a power of 2, at least 2. It generates the grid-level warp table by sampling on the pixel-level warp table directly. The bigger grid spacing takes the less running time.

Parameters
paramsTHe warp parameters.
deviceA type in ea_device_e.
Returns
ea_warp_t* The warp context.
